### 3.3 Listening Authenticity Standards
Per CEFR Companion Volume (2020) listening construct:
- **Speech rate:** 120–150 wpm (A1/A2), 140–165 wpm (B1/B2), 150–180 wpm (C1/C2)
- **TTS settings:** Natural rate 0.90–0.95 (not artificially slow)
- **Accent variation:** rotate BrE / AmE / AusE (see `TTS_VOICES` array in script)
- **Text types:** dialogues, monologues, announcements, interviews, lectures, podcasts
- **No verbatim overlap:** listening text and question stem must not share >3 consecutive words (tests inference, not verbatim matching)

**Writing:** Task Achievement | Coherence & Cohesion | Lexical Resource | Grammatical Range & Accuracy (0–5 each, total 20)

**Speaking:** Grammatical Range & Accuracy | Lexical Resource | Fluency & Coherence | Pronunciation (0–5 each, total 20)

Band descriptors must be cross-referenced against CEFR Companion Volume 2020 table for the target level.

## 4. Quality Assurance Protocol

### 4.1 Three-Persona AI Pipeline (implemented in `ai-item-generator.ts`)
```
WRITER (gemini-2.5-flash, temp=0.7)
  → generates item with CEFR knowledge block injected
REVIEWER (gemini-2.5-flash, temp=0.3)
  → critiques: construct validity, CEFR alignment, bias, IRT plausibility
REVISER (gemini-2.5-flash, temp=0.5)
  → fixes all critical/major issues before saving
```

### 4.2 Automated QA Gates (all must pass before PRETEST status)
1. **IRT parameter range:** a ∈ [0.70, 2.00], b within ±1.2 logits of CEFR target, c ∈ [0.00, 0.25]
2. **Readability gate:** Flesch-Kincaid grade level within CEFR band tolerance (`readability-engine.ts`)
3. **Lexical standard:** stimulus vocabulary within Oxford 3000/NGSL frequency band for level (`vocabulary-profiler.ts`)
4. **Instruction leakage check:** no meta-text in stems (`audit-item-quality.ts` patterns)
5. **All-caps check:** no unlisted uppercase words (ALLOWED_CAPS whitelist)
6. **Minimum QA score:** ≥ 55/100 overall (configurable via `MIN_QA_SCORE`)

### 4.3 Human Expert Review Triggers
Items routed to expert review queue if ANY of:
- QA score 55–69 (borderline)
- Reviewer verdict = REVISE with ≥2 major issues remaining post-revision
- `cefrAlignmentScore < 3` from three-persona pipeline
- Content flagged for potential bias by reviewer

### 4.4 Activation Criteria (PRETEST → ACTIVE)
From `item-generation-pipeline.ts` `ACTIVATION_CRITERIA`:
- Minimum 50 real examinee responses
- Point-biserial r_pb ≥ 0.30
- Outfit MNSQ ≤ 2.0, Infit MNSQ ≤ 1.5
- p-value ∈ [0.10, 0.95]
- No Lord χ² drift flag (Class A/B only — Class C → auto-retired)

### 4.5 DIF Screening
Post-pretest, each item screened for Differential Item Functioning:
- Mantel-Haenszel Δ ≤ |1.5| (Class A — no action)
- MH Δ ∈ (1.5, 2.5] (Class B — flagged, expert review)
- MH Δ > 2.5 (Class C — retired immediately)
- Subgroup splits: gender, L1 region (if available), age group

## 9. Validity Argument

This expansion is grounded in the following validity claims (Bachman & Palmer, 2010):

1. **Construct representativeness:** each CEFR level has ≥40 items per skill → sufficient sampling of the construct space for reliable CAT estimation (CSEM ≤ 0.35 at all θ values)

2. **Item authenticity:** topics, text types, and task demands match real-world language use documented in CEFR Companion Volume (2020) can-do descriptors

3. **Score comparability:** all new items share the same IRT scale as existing items via pretest embedding → equating not required (concurrent calibration)

4. **Fairness:** three-persona AI pipeline includes explicit bias review; DIF screening post-pilot; topic diversity matrix prevents systematic cultural loading

5. **Reliability:** target Cronbach's α ≥ 0.88 for 30-item adaptive tests (verified via simulation); item-level reliability gates prevent poorly discriminating items from reaching ACTIVE status
